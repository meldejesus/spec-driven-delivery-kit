# Message Style Lessons

Reusable lessons for turning dense technical material into clear technical, mixed-audience, or non-technical communication.

Add to this file only when a lesson is broadly useful across future messages.

## Current Lessons

- Start by naming the audience and what they need to do with the information.
- Preserve technical truth, but choose the lowest useful level of implementation detail.
- Replace internal implementation names with user-facing concepts unless the audience needs the implementation term.
- Use an outline gate before drafting; it is cheaper to fix structure before prose exists.
